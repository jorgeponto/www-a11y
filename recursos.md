### Agentes de Utilizador {#agentesdeutilizador}

Para aceder à informação, os utilizadores precisam de software, tecnicamente designado por agente de utilizador.

#### Leitores de PDF {#au_leitoresdepdf}

  * [<abbr title="Visually Impaired People" lang="en">VIP</abbr> PDF](http://www.szb.ch/en/press/press-releases/the-first-pdf-reader-for-visually-impaired-people.html) | Leitor PDF para utilizadores com baixa visão, elaborado pela Associação dos Cegos da Suiça. Este leitor de PDF fornece de imediato informação sobre se estamos perante um PDF que segue ou não as regras de acessibilidade.

### Ferramentas de Autor {#ferramentasautor}

O conhecimento das **linguagens de programação** e das **ferramentas que as manuseiam e produzem** são um elemento fundamental para a obtenção de um bom resultado por parte de todos aqueles que se dedicam à concepção de conteúdos digitais.

#### Ferramentas de Edição Web {#n1}

  * [Funcionalidades de acessibilidade no <span lang="en" xml:lang="en">FrontPage</span>](http://office.microsoft.com/en-gb/assistance/HP030894251033.aspx?Query=accessibility&Scope=TC).
  * [Dreamweaver e outros produtos de edição Adobe](http://www.adobe.com/accessibility/).

#### Acessibilidade em Línguagens e Formatos não <abbr title="World Wide Web Consortium" lang="en">W3C</abbr> {#n2}

  * **<span lang="en" xml:lang="en">Flash</span> e acessibilidade**. 
      * _Best Practices for Accessible Flash Design_ 
          * This whitepaper, written by Bob Regan, describes the framework to approach accessible design using Macromedia Flash. This document will help designers and developers to understand and implement accessibility best practices in their web applications.
          * [Best Practices for Accessible Flash Design (Flash, 234K)](http://www.macromedia.com/resources/accessibility/best_practices/best_practices_acc_flash.swf) | [Best Practices for Accessible Flash Design (PDF, 234K)](http://www.macromedia.com/resources/accessibility/best_practices/best_practices_acc_flash.pdf).
      * _Creating Accessible Macromedia Flash Content_ 
          * Contents: (1) Flash Accessibility Overview; (2) Assistive Technology Support for Flash; (3) Screen Reader Accessibility.
          * [Creating Accessible Macromedia Flash Content (HTML version)](http://www.webaim.org/techniques/flash/).
  * **<abbr title="Portable Data Format" lang="en">PDF</abbr> e acessibilidade.** 
      * _Documentos da Adobe sobre Acessibilidade_. 
          * [Ler conteúdos PDF](http://www.adobe.com/enterprise/accessibility/readcontent.html) | [Criar conteúdos PDF acessíveis](http://www.adobe.com/enterprise/accessibility/creating.html) | [Converter conteúdos PDF](http://www.adobe.com/products/acrobat/access_onlinetools.html).
      * _Adobe Acrobat Accessibility Techniques._ 
          * Contents: **(1)** Introduction; Mobility impairments; Hearing impairments; Cognitive impairments; Low vision; Blindness and screen reader accessibility; **(2)** A Closer Look at What Developers Must Do; The document must contain real text; The document must be in Tagged PDF format; The document must be marked up for accessibility; Guidelines for all disability types; (3) Conclusion.
          * [Adobe Acrobat Accessibility Techniques (HTML version)](http://www.webaim.org/techniques/acrobat/).
  * **<span lang="en" xml:lang="en">JavaScripts</span> e acessibilidade**. 
      * _Java Documentation and Specs_. 
          * Links to Java technology and specifications.
          * [Ver Java Documentation and Specs](http://www.sun.com/access/general/javadocs.html).
      * _Java Accessibility Quick Tips._ 
          * Quick tips on how to make Java applications accessible.
          * [Ver Java Accessibility Quick Tips](http://www.sun.com/access/developers/tips.html).
      * _The Java Tutorial &#8211; Accessibility section._ 
          * Online tutorial on how to use various Swing features.
          * [See The Java Tutorial &#8211; Accessibility section](http://java.sun.com/docs/books/tutorial/uiswing/misc/).
      * _Java Accessibility Guidelines_. 
          * Techniques pages, accessed via the link in each checkpoint, may contain more recent updates than the checklist itself. Be sure to review the techniques pages for the latest accessibility guidance.
          * [See Java Accessibility Guidelines](http://www-3.ibm.com/able/guidelines/java/accessjava.html).
      * _Jared Smith. (July 2004). Creating Accessible JavaScript._ 
          * Content: What is JavaScript?; JavaScript Accessibility Issues ; JavaScript that does not impact accessibility; Comparison of JavaScript Guidelines; Testing for JavaScript Reliance; Disabling JavaScript; Self-check Questions.
          * [See Creating Accessible JavaScript (HTML version)](http://www.webaim.org/techniques/javascript/).
  * **<span lang="en" xml:lang="en">Oracle</span> e acessibilidade.** 
      * _(August 2004). Accessibility in Oracle Forms Applications &#8211; An Oracle White Paper._ 
          * Table of Contents: 1. Introduction; 2. Running a form if you are a disabled user; 3. Building forms for disabled users; 4. Versions and known issues; 5. Conclusion.
          * [See the Oracle White Paper &#8211; Accessibility in Oracle Forms Applications](http://www.oracle.com/accessibility/apps02.html).
